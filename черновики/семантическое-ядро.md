# Semantic Core — Pickpock (MVP)

Pickpock — это **personal pocket memory**.  
Каждая запись — это **событие**, зафиксированное как комбинация действия, объекта и контекста.

---

## 1. Canonical Formula

```

Entry = Action + Object + Context

```

- **Action** — что произошло или что планируется
- **Object** — с чем связано действие
- **Context** — дополнительные детали (кто, где, когда, сколько, фото и т.д.)

---

## 2. Actions (fixed MVP set)

### Memory & Intent
- `noted` — just note / remember
- `visited` — visited / was at
- `planned` — want to / plan to

### Money
- `spent` — spent money
- `received` — received money
- `lent` — lent money
- `borrowed` — borrowed money
- `returned` — returned / was returned

### Items
- `bought` — bought item
- `gave` — gave / gifted item
- `took` — took / received item

**Rule:** exactly **one action per entry**.

---

## 3. Objects

- `money` — money
- `place` — place / location
- `food` — food / dish
- `item` — item / thing
- `service` — service
- `wish` — wish / goal
- `task` — task / errand
- `note` — generic note (fallback)

---

## 4. Context (optional)

### Common
- `title`
- `date`
- `status` (`active | done | waiting`)
- `tags[]`
- `attachments[]` (photo, receipt, screenshot)

### People & Location
- `people[]`
- `placeText`
- `geo`

### Financial (when applicable)
- `amount`
- `currency`
- `category`
- `source`

### AI
- `aiRaw`
- `confidence`

---

## 5. Human Examples

- “Went to cafe ABC” → `visited + place`
- “Want to go to cafe ABC” → `planned + place`
- “Ate carbonara” → `noted + food`
- “Spent 650 on taxi” → `spent + money`
- “Lent Pete 1000” → `lent + money`
- “Took a drill from Maria” → `took + item`
- “Remember a lamp in the shop” → `noted + item`

---

## 6. Core UX Principle

If unsure:
```

noted + note

```

Everything else can be уточнено later.

---

## 7. Product Positioning

Pickpock is:
- not accounting
- not notes
- not task manager

Pickpock is **memory of life events**,  
where money, places, items and plans are just facets of the same thing.

---

## 8. Stability Rule

This semantic core is:
- minimal
- complete
- extensible

New objects can be added without breaking existing data or logic.
