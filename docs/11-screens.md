# 11 — Экраны (по скриншотам)

Ниже — консолидированное описание экранов, собранное из имеющихся скриншотов (PickPock / Banani). Это не финальный UI, а ориентир по структуре, смыслам и полезным блокам.

## 1) Главный экран (Dashboard / Home)

**Что видим:**

- приветствие и короткий статус дня (число записей, черновики, синхронизация);
- быстрый поиск/чат с ИИ;
- быстрые действия: новая запись, скан чека, голос;
- «фокус недели» или инсайты по привычкам/расходам;
- быстрые категории/чипы для фильтрации;
- карточка «важное сейчас» (ближайшее напоминание/задача);
- последние записи и ссылка «показать все».

**Зачем это нужно:**

- сразу понятно, что делать (ввод/поиск/чат);
- есть «момент пользы» без лишних действий (инсайты);
- быстрый доступ к истории.

### Варианты дашборда (из мокапов)

- **Compact/Mobile‑вариант**: минимум блоков, акцент на «что сделать сейчас».
- **Rich/Detailed**: дополнительные инсайты, графики/сводки, недельные тренды.

**Зачем это нужно:**

- можно варьировать насыщенность в зависимости от стадии продукта и аудитории;
- «богатая» версия подходит для аналитики, компактная — для быстрого ввода.

## 2) История / Все записи

**Что видим:**

- список карточек с иконкой, названием, суммой/типом, датой;
- фильтры по периоду, категориям, эмоциям;
- чипы действий/подтипов (например, «потратил», «получил», «вернули»);
- быстрый поиск по строке;
- FAB для добавления.

**Польза:**

- быстрый просмотр событий;
- сортировка по смыслу (категории, эмоции, временные фильтры).

## 3) Карточка записи (Entry Details)

**Что видим:**

- заголовок и сумма/ключевое поле;
- дата/время;
- категория, человек, место, теги;
- вложения и заметки;
- действия: редактировать, напоминание, поделиться.

**Польза:**

- простой просмотр контекста;
- быстрое редактирование без потери смысла.

## 3.1) AI‑предпросмотр (AI Preview)

**Что видим:**

- распознанный смысл записи и ключевые поля;
- предупреждения о низкой уверенности;
- быстрые правки до сохранения.

**Польза:**

- уменьшает ошибки ИИ и ускоряет подтверждение.

## 3.2) Списки (List)

**Что видим:**

- экран списка с набором записей (например, позиции чека);
- сумма/итог по списку;
- быстрые действия: добавить запись, распознать ещё.

**Польза:**

- удобно работать с «пачками» событий;
- меньше кликов при разборе чеков и наборов.

## 4) Поиск и AI‑чат

**Что видим:**

- строка поиска с подсказками;
- блок AI‑запросов («кофе за неделю», «транспорт за месяц»);
- чат‑режим с ответом и ссылками на записи;
  - список найденных карточек;
  - уточняющие вопросы/шорткаты для фильтра.

**Польза:**

- поиск «по‑человечески»;
- быстрое получение инсайтов и ответов.

## 5) Быстрое добавление (Quick Add)

**Что видим:**

- экран выбора категории или типа ввода;
- кнопка сканирования фото/чека;
- быстрый выход/закрытие;
  - быстрые варианты: текст, ссылка, категории;
  - отдельный шаг для выбора «намерения» при фото/скрине.

**Польза:**

- ускоряет фиксацию событий без заполнения формы.

## 6) New Entry Wizard — шаги

### Шаг 1: Ввод
- текст, фото, ссылка, голос;
- краткая подсказка, что можно ввести.

### Шаг 2: AI‑черновик
- распознанные поля (action/object, сумма, место, человек);
- показатель уверенности;
- возможность поправить поля;
  - список типов записи (для категории «деньги»: потратил/получил/вернули и т.д.).

### Шаг 3: Детали и подтверждение
- редактируемые поля (сумма, категория, дата);
- эмоции/настроение;
- сохранение записи;
  - вложения/фото и длинное описание (если есть распознавание).

**Польза:**

- ИИ берет на себя структуру;
- пользователь контролирует финальный результат.

### Экран захвата (New Entry Capture)

- фокус на вводе: камера/текст/ссылка;
- минимальные элементы управления, чтобы не отвлекать от фиксации.

### Экран ревью (New Entry Review)

- быстрый просмотр ключевых полей;
- кнопка «исправить» и доступ к полному редактированию.

## 7) Настройки, предпочтения, напоминания

**/settings**

- базовые настройки (тема, валюта);
- экспорт данных;
- управление приватностью;
  - включение/выключение ИИ и выбор модели анализа.

**/preferences**

- включение/выключение категорий и рецептов;
- управление полями и кастомными сценариями;
  - управление своими рецептами (пустое состояние + «создать»).

**/reminders**

- список активных напоминаний;
- изменение/отключение по записи.

## 7.1) Группы и совместные пространства

**/groups**

- список групп (семья/команда);
- общий и приватный разделы;
- шаринг отдельных записей/списков.
- приглашение участников и управление ролями.

## 8) Планировщик

**/planner**

- календарь/расписание ближайших напоминаний;
- фильтры по категориям и типам событий;
- быстрый переход к карточке события или редактирование даты.

## 9) Онбординг

**/onboarding**

- приветственный экран;
- короткое объяснение смысла приложения;
- первая запись как пример.

### Onboarding Welcome (из мокапов)

- один главный тезис о ценности;
- кнопка «сделать первую запись».

## 9.1) Дизайн‑система (Design System)

**Что видим:**

- набор базовых компонентов (кнопки, чипы, карточки, поля);
- типографика и цветовые роли;
- состояния (loading/empty/error).

**Польза:**

- ускоряет сборку новых экранов;
- снижает визуальный разброс.

## 10) Полный список маршрутов (для сверки)

- `/` — история (лента): недавние записи, быстрые действия, инсайты, «важное сейчас».
- `/new` — визард создания: пошаговый ввод → AI‑черновик → подтверждение.
- `/enter` — быстрый ввод: минимальный экран для текста/голоса/ссылки/скана.
- `/entries` — список/фильтры: карточки, фильтры, быстрый поиск.
- `/entry/:id` — карточка события: поля, вложения, действия.
- `/list/:id` — список записей: позиции чека или группа записей.
- `/search` — поиск: строка + подсказки.
- `/bot` — AI‑чат: диалог, выдача и уточнения.
- `/lifeline` — линия жизни: ключевые события по периодам.
- `/settings` — настройки: базовые параметры, экспорт, приватность.
- `/preferences` — предпочтения: категории, рецепты, поля.
- `/reminders` — напоминания: список активных и настройки.
- `/planner` — планировщик: календарь напоминаний/намерений.
- `/groups` — совместные пространства: группы, общий/приватный контур.
- `/dashboard` — сводка/аналитика: после MVP.
