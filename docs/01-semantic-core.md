# 01 — Семантическое ядро

## Базовая формула

**Событие = Действие + Объект + Контекст**

- **Action (Действие)** — что произошло или планируется.
- **Object (Объект)** — с чем связано действие.
- **Context (Контекст)** — детали (кто, где, когда, сколько, фото, эмоции и т.д.).

Контекст опционален. Action и Object — обязательны и должны быть однозначны.

## Набор действий (MVP)

### Память и намерения
- `noted` — просто заметил/запомнил
- `visited` — был/посетил
- `planned` — хочу/планирую

### Деньги
- `spent` — потратил
- `received` — получил
- `lent` — одолжил
- `borrowed` — занял
- `returned` — вернул/вернули

### Вещи/объекты
- `bought` — купил
- `gave` — подарил/отдал
- `took` — взял/получил

**Правило:** одно событие = одно действие.

## Набор объектов (MVP)

- `money`
- `place`
- `food`
- `item`
- `service`
- `wish`
- `task`
- `note` (fallback)

### На рассмотрение: эмоции как объект

Эмоции можно фиксировать через контекст (`mood`, `emotions`) без расширения ядра. Если нужна «killer‑фича» про память чувств, можно рассмотреть объект `emotion`, но это расширение ядра и требует отдельного решения.

## Примеры

- «Съел карбонару» → `noted + food`
- «Потратил 650 на такси» → `spent + money`
- «Хочу сходить в кафе» → `planned + place`
- «Одолжил Пете 1000» → `lent + money`
- «Взял дрель у Марии» → `took + item`

## Правило неопределенности

Если пользователь или ИИ не уверены, применяется безопасный fallback:

**`noted + note`**

Детализация может происходить позже.

## Стабильность ядра

- Семантическое ядро минимально и редко меняется.
- Новые категории вводятся через рецепты, а не через расширение ядра.
- Контекст может расширяться без изменения действий и объектов.
